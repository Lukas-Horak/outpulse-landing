00 · Overview

Principles

Three constraints the system is built around. When a component or screen doesn't feel right, it almost always violates one of them.

01

Quiet surface, loud signal.

Backgrounds, borders, and text default to neutral tones. Colour enters only when it carries meaning — status, signal, outcome. Never decoration.

02

Serif weight, sans workload.

Instrument Serif for the one display line that earns attention. Inter for every table, label, and paragraph. The switch between them is the signal that the role of the text changed.

03

Data dense, not busy.

A card should answer the question before the reader scrolls. KPIs lead. Trends sit under them. Filler chrome — gradients, illustrations, emoji — is removed unless it resolves a specific confusion.

Semantic tokens

Don't reach for raw colour values in a component. Use the semantic token that describes the role — if the system changes how it expresses success, you only change it once.

SuccessVerified emails · running campaigns · winning flagvar(--green)
Info · progressSending · info banners · in-progress statevar(--blue)
Attention · primary actionUnread · paused · CTA hovervar(--orange)
WarningApproaching quota · missing datavar(--warn)
Error · destructiveBounced email · delete action · validation errorvar(--red)
AI · generatedAI insight banners · persona chips · product workspacevar(--violet)
Do
Use --green for anything that means "working as intended." Use --orange sparingly — one action per screen.
Don't
Never pair Ember and Amethyst inside the same component — it reads as decorative, not semantic.
02 · Foundations

Typography

Instrument Serif is the display face — one Playfair-replacement per page, used for hero headlines and KPI numbers. Inter runs everything else. JetBrains Mono is reserved for tokens, code, and tabular numbers where alignment matters.

Do
Switch from serif to sans when the text switches from "hero statement" to "working copy." Never mix them in the same line.
Don't
Don't italicise serif headlines except for a single em accent per page — the eye stops believing the emphasis once it sees it twice.
03 · Foundations

Spacing, radius, shadow

Base-4 scale for everything. Radius expands with container size. Shadow is used sparingly — the product is mostly flat, elevation is reserved for cards that really do lift.

Spacing · base 4

04 · Foundations

Motion

Three durations, one easing. Everything animated in the product uses cubic-bezier(.4, 0, .2, 1) — tight, material-adjacent, no overshoot. Hovers are fast, state changes are base, layout shifts are slow.

--motion-fast120ms · hover states, chip-click feedback
--motion-base200ms · focus rings, reveal transitions, dropdown
--motion-slow350ms · layout shifts, progress bars, skeletons
/* Always use the same easing curve. */
--motion-ease: cubic-bezier(.4, 0, .2, 1);
--motion-fast: .12s var(--motion-ease);
--motion-base: .2s var(--motion-ease);
--motion-slow: .35s var(--motion-ease);
05 · Components

Icons

Lucide-style stroke set — 24×24 viewBox, 2px stroke, rounded caps + joins. Stroke icons are the default; fill-glyphs (kebab, play, pause, LinkedIn, brand tags) carry their own semantics. Never tint icons unless the colour is part of the meaning.

Use via <use href="#i-name"/> inside an inline <svg>. All symbols live in one block at the top of the page.

Four levels · three sizes · five states. Primary is ink — one per screen, hovers to ember to reward the click. Ghost is neutral. Danger is reserved for destructive actions. Icon buttons are 36×36, FAB is a 56×56 circle for mobile.
