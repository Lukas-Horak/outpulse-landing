### Auto-responder template (on `lukas@outpulse.ai`)

Set this as a Gmail vacation responder (or equivalent) so every reply gets a single, identical, fully-templated response. Gmail's vacation responder only sends once per 4 days to the same sender, which is ideal — no runaway loops.

> **Subject:** Thanks for replying — a quick note from the robot
>
> Thanks for replying.
>
> To keep my promise — I'm not in this inbox. Outpulse (the tool that sent you the email) is fully self-serve, and you can try it free for 7 days at **outpulse.ai**.
>
> If you hit a bug or something breaks, drop a line to **hello@outpulse.ai** and a human will answer within a day.
>
> If this is a partnership, press, or investor inquiry, reach me directly at **lukas@plat4m.com**.
>
> — Lukas (via robot)

The auto-responder has three escape hatches by design: product issues route to `hello@`, partnerships route to `plat4m.com`, everything else resolves itself. You never open this inbox.

---

### STOP handling

When a recipient replies with "STOP" (or the auto-detected variants: `unsubscribe`, `remove me`, `take me off`, `do not contact`), Outpulse must:

1. Immediately suppress the contact from the current campaign
2. Add them to the global suppression list (never send from any future campaign)
3. Not trigger the auto-responder on the STOP reply itself (they've opted out)

**Before launch — verify the app supports this.** If not, this is a blocking feature and must ship before the dogfood campaign goes live. It's also non-negotiable for client campaigns (CAN-SPAM + GDPR).

## Visual assets

### Primary — LinkedIn native video (STRONGEST, use this if possible)

**Source:** `onboarding.html` — the interactive onboarding flow. Each section auto-animates its own demo: Company auto-types "PLATFORM" + "www.plat4m.com" with a "Scanning website… Analyzed ✓" badge, ICP chips appear, email composer writes itself, dashboard fills in with prospects. End-to-end ~60–90 seconds of product-walking-itself-through-onboarding.

Why this beats any image: LinkedIn native video pulls ~3x the impressions and engagement of image posts, and this specific video matches the narrative of the launch post perfectly — the product is literally setting itself up in front of the viewer.

**Recording steps (10 min total):**

1. Open `onboarding.html` in Chrome or Safari at viewport **1440×900** (or **1280×800** for smaller file). Use fullscreen / hide-toolbar (`⇧⌘F` in Chrome) to remove browser chrome.
2. Open QuickTime → **File → New Screen Recording** (or `⇧⌘5`). Set capture region to the browser window. **Disable "Record cursor"** in QuickTime options if you want a clean recording; otherwise move the mouse slowly and deliberately.
3. **IMPORTANT: refresh the page (`⌘R`) before starting recording.** The demo functions have a `demoTriggered` flag in the code — each section only auto-animates once per page load. If you mess up the recording and need a retake, you MUST refresh first, otherwise the already-seen sections stay in their final state without animation.
4. Start recording, then click/scroll through all 5 sections: Welcome → Company → ICP → Email → Done → Dashboard. Let each section fully finish its animation before moving to the next (~3–5 s pause per section).
5. Stop recording. Target length: **60–90 seconds** (optimal for LinkedIn).
6. Export: QuickTime saves as `.mov`. For a smaller file, run:
   ```
   ffmpeg -i recording.mov -vcodec h264 -acodec aac -b:v 2M -movflags +faststart outpulse-launch-video.mp4
   ```
7. Upload directly to LinkedIn as native video (NOT as a YouTube link — native video gets the algorithmic boost).

**Optional second cut — square (1:1) for X / Instagram:**
Re-record at 1080×1080 viewport, or crop the 16:9 recording to square in QuickTime / ffmpeg:
```
ffmpeg -i outpulse-launch-video.mp4 -vf "crop=ih:ih" -c:a copy outpulse-launch-video-square.mp4
```
